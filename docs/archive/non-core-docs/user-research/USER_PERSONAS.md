# 用戶角色定義

**專案**：鴿子競賽 GPS 追蹤系統
**最後更新**：2025-11-18
**版本**：1.0

---

## 📖 目的

本文檔定義鴿子競賽 GPS 追蹤系統的核心用戶角色，包括其身份特徵、使用目標、技能水平、關鍵需求和痛點。這些用戶角色將指導：
- 功能優先級決策
- 用戶界面設計
- 測試案例設計
- 用戶體驗優化

---

## 🎯 主要用戶角色

### 角色 1：賽鴿愛好者 / 參賽者（鴿主）

#### 身份描述
- **身份**：鴿主、參賽者
- **背景**：擁有並管理一個或多個鴿舍，參加各類賽鴿比賽
- **使用頻率**：賽事期間高頻使用（每週數次），非賽事期間偶爾查看歷史記錄
- **技術熟悉度**：中等（能使用網頁瀏覽器和基本操作）

#### 使用目標

##### 主要目標
1. **即時追蹤比賽進度** - 在賽事進行中查看自己鴿子的飛行軌跡和實時位置
2. **查看比賽成績和排名** - 確認鴿子的最終名次和飛行成績
3. **分析飛行數據** - 研究鴿子的飛行速度、高度、距離等數據，用於訓練改進
4. **比較多只鴿子表現** - 對比自己不同鴿子在同一賽事或不同賽事中的表現

##### 次要目標
- 管理鴿舍資訊（查看鴿舍內所有鴿子）
- 通過環號快速找到特定鴿子
- 了解其他參賽者的成績（學習和對標）
- 保存歷史記錄供日後參考

#### 技能水平
- **網頁操作**：熟練（日常使用電腦和手機瀏覽器）
- **地圖應用**：中等（使用過導航軟體，但可能不熟悉 3D 地圖）
- **數據分析**：基礎到中等（能理解速度、距離等基本指標）
- **賽鴿專業知識**：專家級（深入了解賽鴿運動規則和術語）

#### 關鍵需求

##### 功能需求（按優先級）
| 優先級 | 需求 | 重要性說明 |
|--------|------|------------|
| P0 | 查看鴿子飛行軌跡 | 核心功能，是使用系統的主要原因 |
| P0 | 查看排名和成績 | 比賽最重要的結果資訊 |
| P0 | 通過環號搜尋鴿子 | 快速定位自己的鴿子（鴿主通常記得環號） |
| P1 | 查看詳細飛行數據 | 用於分析和改進訓練方案 |
| P1 | 切換 2D/3D 視圖 | 不同情境下的視覺偏好 |
| P1 | 管理鴿舍列表 | 組織和管理多只鴿子 |
| P2 | 動態播放飛行過程 | 增強視覺體驗，但非必需 |
| P2 | 比較多只鴿子數據 | 進階分析需求 |

##### 非功能需求
- **速度**：頁面載入和地圖渲染需要快速（<3 秒）
- **可靠性**：數據準確性至關重要（GPS 軌跡、時間、速度）
- **易用性**：介面直觀，無需閱讀說明文檔即可使用核心功能
- **可訪問性**：支援手機和桌面瀏覽器
- **視覺清晰度**：軌跡線條、標記點清晰可見

#### 典型使用場景

##### 場景 1：比賽當天查看成績（高頻場景）
**時間**：比賽結束後數小時內
**目標**：確認自己鴿子的名次和飛行表現
**步驟**：
1. 進入網站首頁
2. 在賽事列表中找到今天的比賽
3. 點擊「進入」查看賽事詳情
4. 通過環號搜尋自己的鴿子（可能有多只）
5. 勾選鴿子，點擊「查看軌跡」
6. 查看軌跡地圖和飛行數據
7. 檢查排名和關鍵指標（分速、距離、高度）

**期望結果**：快速找到鴿子，清楚看到軌跡和成績

##### 場景 2：賽後深度分析（中頻場景）
**時間**：比賽結束後數天內
**目標**：分析鴿子的飛行策略，找出改進空間
**步驟**：
1. 進入賽事詳情頁
2. 選擇自己的一只或多只鴿子
3. 切換 2D/3D 視圖比較
4. 查看軌跡點詳細數據（每個時間點的速度、高度）
5. 對比不同鴿子的飛行路線差異
6. 記錄觀察結果

**期望結果**：獲得詳細的飛行數據用於訓練優化

##### 場景 3：管理鴿舍內所有鴿子（低頻場景）
**時間**：賽季開始前或賽季中期
**目標**：查看自己鴿舍的所有參賽鴿子
**步驟**：
1. 進入賽事詳情頁
2. 切換到「鴿舍列表」標籤
3. 選擇自己的鴿舍編號
4. 查看該鴿舍內所有鴿子的排名
5. 選擇特定鴿子查看詳情

**期望結果**：一覽自己所有鴿子的表現

#### 痛點和挑戰

##### 當前系統的痛點（基於已知問題）
| 痛點 | 影響程度 | 已知解決方案 |
|------|----------|--------------|
| 2D 模式初次載入失敗 | 高 | 使用 3D→2D 切換順序（docs/guides/troubleshooting.md） |
| 靜態/動態模式混淆 | 中 | 通過標記點數量判斷（15-20 vs 1-3） |
| 軌跡點點擊無響應 | 中 | 使用 accessibility tree locator |
| 數據載入時機不明確 | 低 | 在切換後等待 2-3 秒 |

##### 一般使用挑戰
- **賽事列表過多**：難以快速找到特定賽事
- **環號記憶負擔**：用戶需要記住多只鴿子的環號
- **模式按鈕文字混淆**：按鈕顯示「3D」卻進入 3D 模式（用戶可能誤以為是切換到 3D 的按鈕）
- **數據載入等待**：沒有明確的載入進度提示

#### 用戶訪談引用
> "我最關心的就是我的鴿子飛得怎麼樣，有沒有進前十名。"
> ——假設用戶反饋（基於用戶行為推測）

> "如果能一次看到我所有鴿子的成績就好了，不用一個個去找。"
> ——假設用戶需求（基於鴿舍列表功能推測）

## 🎯 次要用戶角色

### 角色 2：賽事觀眾 / 愛好者

#### 身份描述
- **身份**：非參賽者，觀看比賽的愛好者
- **背景**：對賽鴿運動感興趣，但未直接參賽
- **使用頻率**：偶爾查看熱門賽事

#### 使用目標
1. 觀看熱門賽事的比賽過程
2. 查看排名前列鴿子的飛行軌跡
3. 了解賽鴿運動的規則和數據

#### 關鍵需求
- 查看公開賽事列表
- 查看任意鴿子的軌跡（無需登入）
- 簡單直觀的介面（無需深入學習）

## 📝 與測試計劃的關聯

### 針對主要角色（賽鴿愛好者）的測試重點

根據用戶需求優先級，測試案例應覆蓋：

#### P0 功能測試
- **TC-02-001 至 TC-02-006**：2D 靜態軌跡顯示（核心需求）
- **TC-04-001 至 TC-04-006**：3D 模式渲染（核心需求）
- **TC-01-004**：進入賽事詳情（入口功能）

#### P1 功能測試
- **TC-03-001 至 TC-03-008**：靜態/動態模式切換（視覺偏好）
- **TC-06-001 至 TC-06-004**：軌跡點互動（數據分析需求）
- **TC-05-001 至 TC-05-004**：鴿舍列表操作（管理需求）

#### P2 功能測試
- **TC-07-001 至 TC-07-006**：UI 元素驗證（使用體驗）
