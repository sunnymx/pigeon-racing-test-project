## Template Usage Notes

### For Template Users

When creating a new teaching document from this template:

1. **Replace all placeholders**:
   - `[Topic]` → Actual technology name
   - `[language]` → Specific programming language
   - `YYYY-MM-DD` → Actual date
   - All bracketed descriptions

2. **Keep structure consistent**:
   - Always include Overview + Deep Dive + Best Practices + Troubleshooting
   - Maintain the ❌/✅ format for before/after examples
   - Keep line count under 200 lines (split if needed)

3. **Code examples must be**:
   - Executable (include necessary imports)
   - Commented (explain key lines)
   - Realistic (based on actual use cases)

4. **Update Quick Navigation**:
   - Add your new topic to SKILL.md Quick Navigation section
   - Link to your reference file
